# CLAUDE.md

## 项目概述

**项目名称**: 打杂工具箱

**项目目标**: 通过AI辅助，实现工作总结的自动化、结构化和知识积累

**核心功能**:
- 每日工作记录与深度探索
- 新闻监测与政策收集
- 周工作总结生成
- 项目跟踪与关联分析
- 知识库建设（项目知识 + 个人知识）

## 核心规则

### 1. 输入输出分离

| 区域 | 位置 | 说明 |
|------|------|------|
| 用户输入 | `工作记录\` | 用户放入工作碎片，AI只读 |
| 项目跟踪 | `项目库\` | 半自动，用户核查AI输出 |
| 个人知识 | `个人知识库\` | AI自动维护，用户可修正 |
| 工具定义 | `.trae\skills\` | AI定义，用户参考 |
| AI输出 | `输出成果\` | AI输出，用户可编辑 |

### 2. 知识萃取机制

| 来源 | 萃取内容 | 更新位置 |
|------|---------|---------|
| 每日工作 | 项目实体、人物、关联 | `项目库\_项目知识\` |
| 每日工作 | 个人技能、经验、学习 | `个人知识库\` |

### 3. 半成品机制

- AI生成初版（半成品），标注待完善内容
- 用户有时间再补充完善
- 降低用户负担

### 4. 以周为单位

- 每日记录碎片 + 知识萃取
- 周末生成周总结 + 知识汇总
- 周末归档

### 5. 文件命名规则

- 日总结（初版）：`YYYY-MM-DD-summary-draft.md`
- 日总结（最终版）：`YYYY-MM-DD-summary.md`
- 周工作总结：`MM.DD-MM.DD周工作总结.md`
- 新闻监测记录：`YYYY-MM-DD.md`
- 归档文件夹：`YYYY年第X周\`

## 涉及的Skills

### 1. 日工作记录
- **位置**: `.trae\skills\日工作记录\`
- **功能**: 每日碎片化工作记录 + 深度探索 + 知识萃取
- **触发**: "今天做了..."、"记录一下..."
- **输出**: 日总结 + 更新知识库

### 2. 周工作总结
- **位置**: `.trae\skills\周工作总结\`
- **功能**: 周工作总结生成 + 知识汇总
- **触发**: "写周报"、"周工作总结"
- **输出**: 周工作总结 + 更新知识库

### 3. 新闻监测
- **位置**: `.trae\skills\新闻监测\`
- **功能**: 新闻监测工具
- **触发**: "/查询新闻"、"/查看新闻"
- **输出**: 新闻监测记录

### 协同关系

```
工作记录\ → 日工作记录 → 日总结
                │
                └──→ 知识萃取 → 项目库\_项目知识\
                            → 个人知识库\

新闻监测（自动）→ 新闻监测记录

周工作总结 ←── 一周素材 + 知识库
     │
     └──→ 知识汇总 → 更新知识库
```

## 关键要点

1. **输入输出分离**
   - `工作记录\` - 用户输入，AI只读
   - `项目库\` - 半自动，用户核查
   - `个人知识库\` - AI自动维护
   - `.trae\skills\` - 工具定义，AI定义
   - `输出成果\` - AI输出，用户可编辑

2. **知识萃取作为副产品**
   - 知识图谱在日常工作中自动生成
   - 无需专门维护

3. **项目知识 vs 个人知识**
   - 项目知识：项目绑定，客观事实
   - 个人知识：个人绑定，能力沉淀

4. **三个Skills协同**
   - **日工作记录**: 深度探索 + 知识萃取
   - **周工作总结**: 结构化总结 + 知识汇总
   - **新闻监测**: 自动化监测

5. **5个必需内容**
   确保周工作总结的完整性和规范性
