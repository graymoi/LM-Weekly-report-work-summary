## 项目概述

**项目名称**: 打杂工具箱

**项目目标**: 通过AI辅助，实现工作总结的自动化、结构化和知识积累

**核心功能**:
- 每日工作记录与深度探索
- 新闻监测与政策收集
- 周工作总结生成
- 项目跟踪与关联分析
- 知识库建设（项目知识 + 个人知识）
- 深度研究与智能分析
- 任务编排与自动化
- 知识增强与智能检索
- 自我优化与系统进化
- 技能发现与推荐
- 技能评估与分析
- 技能自动生成
- 技能自动进化
- 性能监控与预警
- A/B 测试与验证

## 核心规则

### 1. 输入输出分离

| 区域 | 位置 | 说明 |
|------|------|------|
| 用户输入 | `工作记录\` | 用户放入工作碎片，AI只读 |
| 项目跟踪 | `项目库\` | 半自动，用户核查AI输出 |
| 个人知识 | `个人知识库\` | AI自动维护，用户可修正 |
| 工具定义 | `.trae\skills\` | AI定义，用户参考 |
| AI输出 | `输出成果\` | AI输出，用户可编辑 |

### 2. 知识萃取机制

| 来源 | 萃取内容 | 更新位置 |
|------|---------|---------|
| 每日工作 | 项目实体、人物、关联 | `项目库\_项目知识\` |
| 每日工作 | 个人技能、经验、学习 | `个人知识库\` |

### 3. 半成品机制

- AI生成初版（半成品），标注待完善内容
- 用户有时间再补充完善
- 降低用户负担

### 4. 以周为单位

- 每日记录碎片 + 知识萃取
- 周末生成周总结 + 知识汇总
- 周末归档

### 5. 文件命名规则

- 日总结（初版）：`YYYY-MM-DD-summary-draft.md`
- 日总结（最终版）：`YYYY-MM-DD-summary.md`
- 周工作总结：`MM.DD-MM.DD周工作总结.md`
- 新闻监测记录：`YYYY-MM-DD.md`
- 归档文件夹：`YYYY年第X周\`

## 关键要点

1. **输入输出分离**

2. **知识萃取作为副产品**

3. **项目知识 vs 个人知识**

   - **知识增强**: 知识检索 + 智能关联 + 知识图谱

   - **技能发现**: 需求分析 + 技能搜索 + 智能推荐 + 技能验证

## 自我优化功能说明

### 自我优化 Skill

实现系统自我进化，提供：

- **使用监控**: 记录规则、skills、钩子的使用情况
- **反馈收集**: 收集使用反馈和问题
- **问题识别**: 自动识别需要优化的点
- **优化建议**: 生成具体的优化建议
- **优化执行**: 配合 Git 执行优化
- **效果评估**: 评估优化效果

**适用场景**:
- 规则优化（使用率、准确性、性能）
- Skills 优化（触发、功能、性能）
- 钩子优化（触发、执行、资源）

### Git 版本控制

配合 Git 实现版本管理和优化实验：

- **分支策略**: main（稳定）、optimize/*（优化分支）
- **提交规范**: [优化类型] 优化内容
- **版本控制**: 完整的历史记录和回滚能力
- **实验优化**: 在独立分支进行优化实验

### 优化评估标准

定义了完整的优化评估标准：

- **规则优化评估**: 使用率、准确性、性能
- **Skills 优化评估**: 触发率、执行效果、性能
- **钩子优化评估**: 触发率、执行效果、性能
- **优化效果评估**: 量化指标、质化指标
- **优化优先级**: P0-P4 优先级矩阵
- **优化风险评估**: 高/中/低风险等级

### 自我优化工作流

```
使用监控 → 反馈收集 → 问题识别 → 优化建议
     ↓
优化执行（Git 分支）→ 自动化测试 → 效果评估
     ↓
用户确认 → 合并到主分支 → 持续改进
```

### 自我进化优势

| 特性 | 说明 |
|------|------|
| **自动化** | 自动监控、识别、建议优化 |
| **数据驱动** | 基于使用数据和反馈 |
| **版本控制** | Git 管理所有变更 |
| **实验安全** | 分支实验，不影响主分支 |
| **持续改进** | 持续监控和优化 |
| **可追溯** | 完整的优化历史记录 |

## 技能生态系统

### 技能发现 Skill

**功能**: 自动发现和推荐新技能，基于用户需求和行业趋势

**核心能力**:
- 需求分析：分析用户工作模式，识别重复性任务
- 技能发现：搜索行业最佳实践，查找开源 skills
- 智能推荐：匹配用户需求，评估技能适用性
- 技能验证：检查兼容性，评估学习成本

**适用场景**:
- 发现新技能需求
- 推荐适合的技能
- 评估技能适用性

### 技能评估 Skill

**功能**: 评估现有技能的效果、性能和适用性

**核心能力**:
- 使用统计：记录使用频率，统计触发次数
- 效果评估：评估输出质量，测量效率提升
- 性能分析：响应时间分析，资源使用监控
- 优化建议：识别优化点，生成优化方案

**适用场景**:
- 评估技能效果
- 分析性能瓶颈
- 生成优化建议

### 技能生成 Skill

**功能**: 基于用户需求和最佳实践，自动生成新的技能

**核心能力**:
- 需求分析：分析用户需求，识别技能类型
- 技能设计：设计技能架构，定义触发条件
- 代码生成：生成技能代码，实现核心功能
- 测试验证：自动化测试，功能验证

**适用场景**:
- 快速创建新技能
- 自动化技能开发
- 批量生成技能

### 技能进化 Skill

**功能**: 实现技能的自动进化，持续优化和改进技能

**核心能力**:
- 进化监控：监控技能使用，收集性能数据
- 进化策略：定义进化目标，选择进化方向
- 进化执行：生成进化版本，测试进化效果
- 进化评估：评估进化效果，对比进化前后

**适用场景**:
- 技能自动进化
- 持续优化技能
- 版本管理

### 性能监控 Skill

**功能**: 实时监控系统性能，提供性能预警和优化建议

**核心能力**:
- 实时监控：监控技能执行，跟踪资源使用
- 性能分析：分析性能趋势，识别性能瓶颈
- 预警通知：性能下降预警，资源超限预警
- 优化建议：识别优化点，生成优化方案

**适用场景**:
- 实时性能监控
- 性能预警通知
- 性能问题诊断

### A/B 测试 Skill

**功能**: 通过 A/B 测试验证优化方案的效果

**核心能力**:
- 测试设计：定义测试目标，设计测试方案
- 测试执行：创建测试版本，分配测试流量
- 数据分析：统计测试数据，分析测试结果
- 结果报告：生成测试报告，提供决策建议

**适用场景**:
- 验证优化方案
- 对比不同版本
- 数据驱动决策

### 技能生态系统协同

```
技能发现 → 技能生成 → 技能评估 → 技能进化
    ↓           ↓           ↓           ↓
发现需求    生成技能    评估效果    进化优化
    ↓           ↓           ↓           ↓
    └───────────→ 技能生态系统 ←──────────┘
                    ↓
               性能监控
                    ↓
               A/B 测试
```

### 技能生态系统优势

| 特性 | 说明 |
|------|------|
| **自动化** | 技能发现、生成、评估、进化全自动化 |
| **数据驱动** | 基于真实数据和反馈 |
| **持续进化** | 技能自动进化，持续优化 |
| **性能监控** | 实时监控，及时预警 |
| **科学验证** | A/B 测试，数据驱动决策 |
