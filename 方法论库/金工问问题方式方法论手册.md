## 🎯 方法论框架

### 一、语言精准原则

> **核心观点**：用标准化语言，不让大模型"误会"

**要点**：
1. 避免模糊表述
2. 使用专业术语
3. 结构化描述
4. 明确边界条件

---

### 二、命令方式

金工的命令方式分为两种：

| 类型 | 特点 | 适用场景 |
|------|------|---------|
| **直给型** | 直接明确，一步到位 | 目标清晰的任务 |
| **刨根问底** | 层层追问，深入本质 | 复杂问题分析 |

---

### 三、需求定位

> **核心观点**：AI能响应需求，也能生成一部分需求，但真正的需求得找准

**需求层次**：

```
第一层：AI能响应的需求（被动）
    ↓
第二层：AI能生成的需求（主动）
    ↓
第三层：人真正需要的需求（核心）
```

**关键判断**：
- 如果自己不知道需求，AI即使生成了需求，那也不是自己要的需求
- AI能生成100个东西，人就用10个
- 你要什么答案，那个答案就是正确的

## 📝 案例示范

### 案例1：场景描述

**金工的表述**：
> 该场景属于微更新、广覆盖场景。其特点是问题真实、需求刚性、单体不大、适用面广，但普遍受产权复杂和财政约束影响，因此更适宜采取"小范围、强功能、低扰动"的复合化路径，而不是大体量地下综合体路径。中央文件明确提出要补齐民生领域公共服务设施短板，完善医疗应急服务体系和公共服务网络，这为此类场景提供了政策支撑。按老旧社区、医院片区、学校周边、景区和公园周边的滚动更新推算，未来5年投资需求预计为0.6-1.0万亿元，未来10年预计为1.2-2.0万亿元。该类投资不会高度集中，但会广泛分布。

**分析**：

| 要素 | 内容 |
|------|------|
| **场景定位** | 微更新、广覆盖 |
| **特点描述** | 问题真实、需求刚性、单体不大、适用面广 |
| **约束条件** | 产权复杂、财政约束 |
| **路径选择** | "小范围、强功能、低扰动"的复合化路径 |
| **政策支撑** | 中央文件明确要求 |
| **数据支撑** | 未来5年0.6-1.0万亿，未来10年1.2-2.0万亿 |
| **分布特征** | 不会高度集中，但会广泛分布 |

**方法论总结**：
1. 先定义场景类型
2. 描述特点（正面+负面约束）
3. 给出路径选择（对比说明）
4. 引用政策支撑
5. 提供数据量化
6. 说明分布特征

## 🔧 标准话术模板

### 模板1：场景描述

```
该场景属于[场景类型]。其特点是[特点描述]，但普遍受[约束条件]影响，因此更适宜采取[路径选择]，而不是[对比路径]。[政策支撑]。按[计算范围]推算，[数据预测]。该类投资[分布特征]。
```

### 模板2：需求确认

```
需求得清晰：
1. [核心需求是什么]
2. [为什么是这个需求]
3. [不是什么需求]

AI能生成100个东西，人就用10个。你要什么答案，那个答案就是正确的。
```

### 模板3：命令精准

```
直给型命令：
[目标] + [约束] + [输出要求]

刨根问底命令：
[现象] → 为什么？ → [原因] → 为什么？ → [本质]
```

## 📊 与AI交互的对比

| 维度 | 普通方式 | 金工方式 |
|------|---------|---------|
| 语言 | 模糊、口语化 | 标准化、精准 |
| 命令 | 随意、不明确 | 直给型/刨根问底 |
| 需求 | 被AI引导 | 主动找准 |
| 结构 | 零散 | 结构化 |
| 效率 | 高耗能语言，系统阻力高 | 精准表达，降低阻力 |

---

## 💡 应用建议

1. **每次与AI交互前**：先问自己"真正的需求是什么？"
2. **使用标准化语言**：避免模糊表述，不让大模型"误会"
3. **选择命令方式**：目标清晰用直给型，复杂问题用刨根问底
4. **验证输出**：AI生成100个，人用10个，你要的答案才是正确的
