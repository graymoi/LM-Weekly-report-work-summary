# 对话日志存档规则

## 一、基本规则

| 规则项 | 内容 |
|-------|------|
| **触发条件** | 每10轮对话（用户+AI为一轮）自动存档 |
| **存档方式** | AI自动存档 |
| **存档格式** | Markdown格式 |
| **文件命名** | `对话日志_YYYY-MM-DD.md`（当日多批汇总为一个文件） |
| **存档位置** | `输出成果\对话日志\YYYY-MM-DD\` |

---

## 二、存档内容结构

每个存档文件包含两部分：

### 2.1 完整对话记录

```markdown
## 对话记录

### 第1轮
**用户**：xxx
**AI**：xxx

### 第2轮
**用户**：xxx
**AI**：xxx
...
```

### 2.2 关键内容摘要

```markdown
## 知识精华

### 核心洞察
- 洞察1
- 洞察2

### 经验总结
- 经验1
- 经验2

### 待解决问题
- 问题1
- 问题2

### 后续行动
- 行动1
- 行动2
```

---

## 四、定期汇总机制

### 4.1 日汇总
- 每日结束时，汇总当日所有对话日志
- 提取当日知识精华
- 更新个人知识库

### 4.2 周汇总
- 每周末，汇总本周所有对话日志
- 提取本周核心经验
- 生成周知识精华报告
- 更新项目库和个人知识库

### 4.3 月汇总
- 每月末，汇总本月所有对话日志
- 提取月度核心经验
- 生成月度知识精华报告
- 归档到知识库

---

## 五、知识精华提取规则

### 5.1 核心洞察
- 对问题本质的深刻理解
- 反直觉的发现
- 跨领域的连接

### 5.2 经验总结
- 可复用的方法论
- 成功的做法
- 失败的教训

### 5.3 待解决问题
- 未完成的任务
- 需要进一步探索的问题
- 遗留问题

### 5.4 后续行动
- 具体的行动步骤
- 优先级排序
- 时间节点

---

## 六、AI职责

1. **对话计数**：跟踪对话轮次
2. **自动存档**：达到10轮时自动存档
3. **内容提取**：自动提取知识精华
4. **定期汇总**：日/周/月汇总
5. **知识更新**：更新项目库和个人知识库
