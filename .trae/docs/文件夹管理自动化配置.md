### 2. Git Hooks 集成

#### Pre-commit Hook

**功能**: 在 Git 提交前自动检查文件夹结构

**触发时机**: 每次 `git commit` 时

**检查内容**:
- 文件夹命名规范
- 文件夹结构一致性
- 非法字符检查

**阻止提交**: 如果发现文件夹结构问题，阻止提交并提示修复

**配置文件**: `.git/hooks/pre-commit`

```powershell
# Git pre-commit hook - 文件夹结构检查
$illegalChars = @('/', '\', ':', '*', '?', '"', '<', '>', '|')
$stagedFiles = git diff --cached --name-only --diff-filter=ACM

foreach ($file in $stagedFiles) {
    $folder = Split-Path -Parent $file
    $folderName = Split-Path -Leaf $folder

    foreach ($char in $illegalChars) {
        if ($folderName -like "*$char*") {
            Write-Host "❌ 错误: 文件夹 '$folderName' 包含非法字符 '$char'"
            Write-Host "💡 建议: 使用 '-' 或 '_' 代替 '$char'"
            Write-Host "📝 示例: 将 'A/B测试' 改为 'AB测试'"
            exit 1
        }
    }
}

Write-Host "✅ 文件夹结构检查通过"
exit 0
```

#### Post-commit Hook

**功能**: 在 Git 提交后生成文件夹结构报告

**触发时机**: 每次 `git commit` 成功后

**生成内容**:
- 文件夹结构快照
- 文件夹命名检查
- 文件夹结构一致性检查

**配置文件**: `.git/hooks/post-commit`

```powershell
# Git post-commit hook - 生成文件夹结构报告
$reportPath = "输出成果\文件夹结构快照_$(Get-Date -Format 'yyyyMMdd_HHmmss').md"
$skillsPath = ".trae\skills"

$report = @"
# 文件夹结构快照

生成时间: $(Get-Date -Format 'yyyy-MM-dd HH:mm:ss')
Git 提交: $(git rev-parse HEAD)

---

## 文件夹列表

"@

$folders = Get-ChildItem -Path $skillsPath -Directory
foreach ($folder in $folders) {
    $report += "`n- $($folder.Name)"
}

$report | Out-File -FilePath $reportPath -Encoding UTF8
Write-Host "✅ 文件夹结构快照已生成: $reportPath"
```

## 使用说明

### 1. 安装 Git Hooks

**步骤**:
1. 将 pre-commit hook 复制到 `.git/hooks/pre-commit`
2. 将 post-commit hook 复制到 `.git/hooks/post-commit`
3. 确保 hook 文件有执行权限

**PowerShell 命令**:
```powershell
# 复制 pre-commit hook
Copy-Item ".trae\hooks\pre-commit" ".git\hooks\pre-commit" -Force

# 复制 post-commit hook
Copy-Item ".trae\hooks\post-commit" ".git\hooks\post-commit" -Force

# 设置执行权限
icacls ".git\hooks\pre-commit" /grant Everyone:F
icacls ".git\hooks\post-commit" /grant Everyone:F
```

### 2. 配置定期检查任务

**步骤**:
1. 打开 Windows 任务计划程序
2. 创建新任务
3. 设置触发器：每周一次
4. 设置操作：运行 `scripts\定期检查文件夹结构.ps1`
5. 保存任务

**PowerShell 命令**:
```powershell
# 创建任务计划程序任务
$action = New-ScheduledTaskAction -Execute "PowerShell.exe" -Argument "-File scripts\定期检查文件夹结构.ps1"
$trigger = New-ScheduledTaskTrigger -Weekly -DaysOfWeek Sunday -At 9am
Register-ScheduledTask -Action $action -Trigger $trigger -TaskName "文件夹结构定期检查"
```

## 故障排除

### 问题1: Git hooks 不生效

**症状**: Git 提交时没有触发 hook

**原因**: Hook 文件没有执行权限或路径错误

**解决方案**:
```powershell
# 检查 hook 文件是否存在
Test-Path ".git\hooks\pre-commit"

# 设置执行权限
icacls ".git\hooks\pre-commit" /grant Everyone:F

# 检查 hook 文件内容
Get-Content ".git\hooks\pre-commit"
```

### 问题2: 定期检查任务不执行

**症状**: 定期检查任务没有按计划执行

**原因**: 任务计划程序配置错误或脚本路径错误

**解决方案**:
```powershell
# 检查任务是否存在
Get-ScheduledTask -TaskName "文件夹结构定期检查"

# 手动运行任务
Start-ScheduledTask -TaskName "文件夹结构定期检查"

# 检查任务历史
Get-ScheduledTaskInfo -TaskName "文件夹结构定期检查"
```

### 问题3: 文件夹管理 Skill 不触发

**症状**: 文件夹相关任务没有自动触发文件夹管理 Skill

**原因**: 任务处理 Skill 配置错误或关键词匹配失败

**解决方案**:
```powershell
# 检查任务处理 Skill 配置
Get-Content ".trae\skills\任务处理\SKILL.md"

# 检查关键词匹配
# 确保文件夹相关关键词在匹配列表中
```
