# 对话日志存档架构改进

## 概述

将对话日志存档功能从单一的 Skill 改进为三层架构：规则 + Hook + Skills，实现职责分离和更好的可维护性。

---

## 改进前

### 架构

```
规则 → Skills
  ↓        ↓
定义    执行
"什么"   "如何"
```

### 问题

1. **缺少触发机制**
   - 规则定义了"每10轮对话自动存档"
   - 但没有明确的触发机制
   - Skills 负责执行，但谁来触发？

2. **职责不清**
   - Skills 既负责触发，又负责执行
   - 违反单一职责原则
   - 难以维护和测试

3. **灵活性低**
   - 触发逻辑硬编码在 Skills 中
   - 难以调整触发条件
   - 难以扩展新的触发方式

---

## 改进后

### 架构

```
规则 → Hook → Skills
  ↓      ↓       ↓
定义   触发    执行
"什么"  "何时"   "如何"
```

### 优势

1. **职责清晰**
   - 规则：定义"应该做什么"
   - Hook：定义"什么时候做"
   - Skills：定义"怎么做"

2. **易于维护**
   - 每层独立，互不影响
   - 修改触发逻辑不影响执行逻辑
   - 修改执行逻辑不影响触发逻辑

3. **易于扩展**
   - 可以添加新的触发方式
   - 可以添加新的执行逻辑
   - 不影响现有功能

4. **易于测试**
   - 可以单独测试 Hook
   - 可以单独测试 Skills
   - 测试成本低

---

## 三层架构详解

### 1. 规则层

**职责**：定义"应该做什么"

**位置**: `.trae\rules\project_rules.md`

**内容**:
```markdown
### 对话日志存档规则

| 规则项 | 内容 |
|-------|------|
| **触发条件** | 每10轮对话（用户+AI为一轮）自动存档 |
| **存档方式** | AI自动存档 |
| **存档格式** | Markdown格式 |
| **文件命名** | `对话日志_YYYY-MM-DD.md`（当日多批汇总为一个文件） |
| **存档位置** | `输出成果\对话日志\YYYY-MM-DD\` |
```

**特点**:
- 静态定义
- 易于理解
- 易于修改

---

### 2. Hook 层

**职责**：定义"什么时候做"

**位置**: `.trae\hooks\对话计数器.md`

**内容**:
```markdown
# 对话计数器 Hook

## 触发条件

- 每次对话后自动触发

## 工作流程

```
对话开始
  ↓
对话计数
  ↓
  ├── 读取计数器
  ├── 更新计数器
  └── 检查是否达到10轮
  ↓
达到10轮？
  ↓
  ├── 是 → 调用对话日志存档 Skill
  │
  └── 否 → 继续对话
          ↓
      等待下一轮对话
```

## 计数器管理

### 计数器位置

```
.trae\data\对话计数器.json
```

### 计数器格式

```json
{
  "count": 0,
  "last_update": "2026-03-07T23:12:00",
  "last_archive": "2026-03-07T23:09:00",
  "archive_count": 1
}
```

### 计数器操作

```powershell
# 读取计数器
Get-Content ".trae\data\对话计数器.json" | ConvertFrom-Json

# 更新计数器
$count = Get-Content ".trae\data\对话计数器.json" | ConvertFrom-Json
$count.count += 1
$count.last_update = Get-Date -Format "yyyy-MM-ddTHH:mm:ss"
$count | ConvertTo-Json | Set-Content ".trae\data\对话计数器.json"

# 重置计数器
$count = Get-Content ".trae\data\对话计数器.json" | ConvertFrom-Json
$count.count = 0
$count.last_archive = Get-Date -Format "yyyy-MM-ddTHH:mm:ss"
$count.archive_count += 1
$count | ConvertTo-Json | Set-Content ".trae\data\对话计数器.json"
```

## 触发逻辑

```powershell
# 每次对话后执行
$countPath = ".trae\data\对话计数器.json"

# 检查计数器文件是否存在
if (-not (Test-Path $countPath)) {
    # 创建计数器文件
    New-Item -ItemType File -Path $countPath -Force | Out-Null
    $count = @{
        count = 0
        last_update = Get-Date -Format "yyyy-MM-ddTHH:mm:ss"
        last_archive = Get-Date -Format "yyyy-MM-ddTHH:mm:ss"
        archive_count = 0
    }
    $count | ConvertTo-Json | Set-Content $countPath
} else {
    # 读取计数器
    $count = Get-Content $countPath | ConvertFrom-Json
}

# 更新计数器
$count.count += 1
$count.last_update = Get-Date -Format "yyyy-MM-ddTHH:mm:ss"

# 检查是否达到10轮
if ($count.count -ge 10) {
    # 触发对话日志存档
    Write-Host "📝 对话日志存档触发：已达到10轮对话" -ForegroundColor Cyan

    # 调用对话日志存档 Skill
    # ... 调用逻辑 ...

    # 重置计数器
    $count.count = 0
    $count.last_archive = Get-Date -Format "yyyy-MM-ddTHH:mm:ss"
    $count.archive_count += 1

    Write-Host "✅ 计数器已重置" -ForegroundColor Green
    Write-Host "📊 累计存档次数: $($count.archive_count)" -ForegroundColor Yellow
}

# 保存计数器
$count | ConvertTo-Json | Set-Content $countPath

Write-Host "🔄 对话计数已更新: $($count.count)/10" -ForegroundColor Gray
```
```

**特点**:
- 动态触发
- 灵活配置
- 易于扩展

### 3. Skills 层

**职责**：定义"怎么做"

**位置**: `.trae\skills\对话日志存档\SKILL.md`

**内容**:
```markdown
# 对话日志存档 Skill

## 概述

实现对话日志存档的具体逻辑，包括创建存档文件、提取知识精华、明确提示用户。

**注意**: 对话计数逻辑已移至 `.trae\hooks\对话计数器.md`，本 Skill 只负责存档执行。

---

## 核心功能

### 1. 创建存档文件
- 创建存档目录
- 创建存档文件
- 写入存档内容

### 2. 提取知识精华
- 提取核心洞察
- 提取经验总结
- 提取待解决问题
- 提取后续行动

### 3. 明确提示用户
- 显示存档信息
- 提供查看链接
- 显示下次存档时间

### 4. 重置计数器
- 重置计数器为0
- 更新存档时间
- 增加存档计数

---

## 触发条件

### 自动触发

- **达到10轮对话**: 由对话计数器 Hook 触发
- **每日结束**: 由对话计数器 Hook 触发
- **手动触发**: 用户手动调用

---

  ├── 提取核心洞察
  ├── 提取经验总结

明确提示用户
  ↓
  ├── 显示存档信息
  ├── 提供查看链接

重置计数器
  ↓
  ├── 重置计数器为0
  ├── 更新存档时间
  └── 增加存档计数
```
```

**特点**:
- 专注于执行
- 逻辑清晰
- 易于测试

## 集成方式

### 方式1：任务处理 Skill 集成（推荐）

**适用场景**：自动触发

**集成位置**：`.trae\skills\任务处理\SKILL.md`

**集成方式**：
```markdown
## 任务处理流程

```
任务接收
  ↓
技能检查
  ↓
技能使用
  ↓
对话计数 ←── 新增
  ↓
  ├── 读取计数器
  ├── 更新计数器
  ├── 检查是否达到10轮
  ├── 达到10轮 → 调用对话日志存档 Skill
  └── 重置计数器
  ↓
输出结果
```
```

**优势**:
- 自动触发
- 无需手动操作
- 集成度高

---

### 方式2：Git Hooks 集成

**适用场景**：Git 提交时触发

**集成位置**：`.git\hooks\post-commit`

**集成方式**：
```powershell
# Git post-commit hook - 对话计数器
# 功能: 每次 Git 提交后更新对话计数器
# 触发: 每次 `git commit` 成功后

# 执行对话计数器 Hook
& ".trae\hooks\对话计数器.md"

exit 0
```

**优势**:
- 与 Git 集成
- 版本控制友好
- 自动化程度高

---

### 方式3：手动触发

**适用场景**：临时触发

**执行命令**：
```powershell
# 执行对话计数器 Hook
powershell -ExecutionPolicy Bypass -File ".trae\hooks\对话计数器.md"
```

**优势**:
- 灵活控制
- 临时使用
- 易于测试

2. **职责分离**: 每层职责清晰
3. **易于维护**: 模块化设计
4. **易于扩展**: 灵活的触发机制

- **职责清晰**: 规则定义"什么"，Hook 定义"何时"，Skills 定义"如何"

- **易于扩展**: 可以添加新的触发方式和执行逻辑
- **易于测试**: 可以单独测试 Hook 和 Skills
