# 会话开始Hook

## 功能说明

在每次会话开始时自动执行，加载必要的上下文信息。

## 触发时机

- 新会话开始时
- TRAE IDE启动时

## 执行内容

### 1. 加载核心知识库（必须执行）

**并行读取以下5个核心文件**：

| 序号 | 文件路径 | 内容说明 |
|------|---------|---------|
| 1 | `项目库\项目跟踪记录.md` | 所有项目状态、进展、待办 |
| 2 | `个人知识库\个人档案.md` | 职业背景、工作习惯、协作偏好 |
| 3 | `项目库\_项目知识\人物库.md` | 项目相关人物信息 |
| 4 | `项目库\_项目知识\项目关联图谱.md` | 项目间关联关系 |
| 5 | `项目库\_项目知识\案例库.md` | 可复用案例经验 |

### 2. 加载辅助知识库（按需执行）

根据用户问题类型，读取以下文件：

| 文件路径 | 读取条件 |
|---------|---------|
| `个人知识库\能力图谱.md` | 涉及能力提升时 |
| `个人知识库\学习记录.md` | 涉及学习内容时 |
| `个人知识库\技能清单.md` | 涉及技能应用时 |
| `个人知识库\经验库.md` | 涉及经验复用时 |
| `项目库\_项目知识\政策库.md` | 涉及政策申报时 |

### 3. 加载最近上下文

读取最近7天的：
- 日总结
- 对话日志精华
- 知识库更新

### 4. 检查外部资源

检查GitHub仓库是否有更新：
- `https://github.com/affaan-m/everything-claude-code.git` (参考仓库)
- `https://github.com/graymoi/LMPROJECT.git` (项目仓库)

## 配置选项

在 `hook-config.yaml` 中配置：

```yaml
session_start:
  enabled: true
  load_personal_profile: true
  load_project_status: true
  load_recent_context: true
  context_days: 7
```

## 注意事项

1. 如果配置文件不存在，使用默认配置
2. 如果个人档案不存在，提示用户创建
3. 如果项目状态文件不存在，跳过该步骤
4. 外部资源检查需要网络连接，失败时跳过
