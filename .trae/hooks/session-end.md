# 会话结束Hook

## 功能说明

在每次会话结束时自动执行，保存会话精华并更新知识库。

## 触发时机

- 会话结束时
- TRAE IDE关闭时
- 用户手动触发

## 执行内容

### 1. 保存会话摘要

生成会话摘要，包含：
- 会话主题
- 主要任务
- 关键决策
- 待处理事项
- 后续行动

### 2. 更新知识库

根据会话内容更新：
- `个人知识库/个人档案.md` - 用户偏好、习惯
- `知识库/技能清单.md` - 新技能
- `知识库/经验库.md` - 新经验
- `知识库/学习记录.md` - 学习内容

### 3. 提取模式

从会话中提取可复用模式：
- 问题解决模式
- 工作流程模式
- 决策模式
- 学习模式

### 4. 生成技能建议

如果发现可复用模式，生成技能建议：
- 技能名称
- 触发条件
- 功能描述
- 优先级

### 5. 更新项目状态

如果涉及项目，更新：
- `项目库/项目跟踪记录.md`
- `项目库/_项目知识/` 相关文件

## 输出格式

```
📋 会话结束处理

📝 会话摘要：
- 主题：[主题]
- 任务：[任务列表]
- 决策：[关键决策]
- 待处理：[待处理事项]

📚 知识库更新：
- 个人档案：[更新内容]
- 技能清单：[新增技能]
- 经验库：[新增经验]

🔍 模式提取：
- 发现模式：X个
- 可复用：X个

💡 技能建议：
- [技能名称] - [优先级]

📁 项目更新：
- [项目名称]：[更新内容]

✅ 会话已保存！
```

## 配置选项

在 `hook-config.yaml` 中配置：

```yaml
session_end:
  enabled: true
  save_session_summary: true
  update_knowledge_base: true
  extract_patterns: true
```

## 自动触发条件

### 触发技能进化

当满足以下条件时，自动触发技能进化：
- 发现新的可复用模式
- 用户反馈效果好
- 使用频率高

### 触发规则更新

当满足以下条件时，建议规则更新：
- 发现新的工作类型
- 发现新的触发条件
- 发现规则冲突

### 触发Hook优化

当满足以下条件时，建议Hook优化：
- Hook执行失败
- Hook性能问题
- Hook触发不准确

## 与其他Skills协同

```
会话结束Hook
    │
    ├──→ 对话日志存档Skill
    │       └──→ 保存完整对话
    │
    ├──→ 技能进化Skill
    │       └──→ 提取模式 → 生成技能
    │
    ├──→ 自我优化Skill
    │       └──→ 分析效果 → 优化建议
    │
    └──→ 知识增强Skill
            └──→ 更新知识库
```

## 注意事项

1. 确保所有文件操作完成后再结束
2. 如果更新失败，记录错误日志
3. 用户确认后再执行重大更新
4. 保留历史版本，支持回滚
