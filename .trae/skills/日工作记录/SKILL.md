# 日工作记录 Skill

## 基本信息

- **名称**: 日工作记录
- **功能**: 每日碎片化工作记录与深度探索 + 知识萃取
- **触发词**: "今天做了..."、"记录一下..."、"日工作记录"
- **关键词**: 日工作记录, 工作日志, 每日反思, 工作记录, 每日总结, 工作碎片, 工作洞察, 每日站会, 工作模式, 工作反思

---

## 功能说明

### 核心功能

1. **碎片记录整理**: 将零散的工作记录整理成结构化内容
2. **深度探索**: 通过启发式提问，帮助用户发现工作中的洞察
3. **知识萃取**: 从日常工作中提取可复用的知识，自动更新知识库

### 知识萃取能力

| 萃取类型 | 识别内容 | 更新位置 |
|---------|---------|---------|
| 项目实体 | 项目编号、项目名称、进展状态 | `项目库\项目跟踪记录.md` |
| 人物实体 | 姓名、单位、职务、关联项目 | `项目库\_项目知识\人物库.md` |
| 项目关联 | 项目间的参考、协同、支撑关系 | `项目库\_项目知识\项目关联图谱.md` |
| 案例经验 | 可复用的项目经验 | `项目库\_项目知识\案例库.md` |
| 政策信息 | 政策名称、类型、关联项目 | `项目库\_项目知识\政策库.md` |
| 个人技能 | 新掌握的技能 | `个人知识库\技能清单.md` |
| 个人经验 | 可复用的经验 | `个人知识库\经验库.md` |
| 学习内容 | 学习项目和知识增量 | `个人知识库\学习记录.md` |

---

## 输入输出

### 输入位置

- **工作记录\260XXX\**: 用户放入的工作碎片（txt文件、项目附件等）

### 输出文件

| 输出类型 | 位置 | 文件命名 |
|---------|------|---------|
| 日总结 | `输出成果\日总结\` | `YYYY-MM-DD-summary-draft.md`（初版）|
| 日总结 | `输出成果\日总结\` | `YYYY-MM-DD-summary.md`（最终版）|

---

## 工作流程

```
步骤1：读取工作碎片
─────────────────────
工作记录\260XXX\ → 读取所有txt和附件

步骤2：整理和结构化
─────────────────────
碎片内容 → 分类整理 → 结构化输出

步骤3：知识萃取（自动）
─────────────────────
结构化内容 → 识别实体 → 提取知识
     │
     ├── 项目实体 → 更新 项目库\项目跟踪记录.md
     ├── 人物实体 → 更新 项目库\_项目知识\人物库.md
     ├── 项目关联 → 更新 项目库\_项目知识\项目关联图谱.md
     ├── 案例经验 → 更新 项目库\_项目知识\案例库.md
     ├── 政策信息 → 更新 项目库\_项目知识\政策库.md
     ├── 个人技能 → 更新 个人知识库\技能清单.md
     ├── 个人经验 → 更新 个人知识库\经验库.md
     └── 学习内容 → 更新 个人知识库\学习记录.md

步骤4：深度探索（可选）
─────────────────────
结构化内容 → 启发式提问 → 发现洞察

步骤5：生成日总结
─────────────────────
结构化内容 + 洞察 + 知识萃取结果 → 日总结（初版）
                                        ↓
                    输出成果\日总结\YYYY-MM-DD-summary-draft.md
```

## 启发式提问框架

### 用于深度探索的问题

1. **关于目标**
   - 今天最重要的事情是什么？
   - 是否达成了预期目标？

2. **关于过程**
   - 哪些做得好？哪些可以改进？
   - 遇到了什么障碍？如何解决的？

3. **关于学习**
   - 今天学到了什么？
   - 有哪些可以复用的经验？

4. **关于下一步**
   - 明天最重要的三件事是什么？
   - 需要什么支持？

## 知识萃取规则

### 项目识别规则

```
识别模式：
- 项目编号：P-YYYY-NNN、T-YYYY-NNN、L-YYYY-NNN
- 项目关键词：项目、工程、改造、建设、谋划

处理方式：
- 新项目 → 添加到 项目跟踪记录.md
- 已有项目 → 更新进展状态
```

### 人物识别规则

```
识别模式：
- 姓名 + 单位/部门
- 姓名 + 职务
- 姓名 + 项目关联

处理方式：
- 新人物 → 添加到 人物库.md
- 已有人物 → 更新关联项目
```

### 关联识别规则

```
识别模式：
- "参考XXX项目" → 案例参考关联
- "支撑XXX" → 知识支撑关联
- "同一地区" → 地区关联

处理方式：
- 新关联 → 添加到 项目关联图谱.md
```
