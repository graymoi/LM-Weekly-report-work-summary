# 任务编排 Skill

## 基本信息

- **名称**: 任务编排
- **功能**: 多步骤任务自动化，借鉴 DeerFlow 的子代理编排能力
- **触发词**: "任务编排"、"自动化任务"、"多步骤任务"、"任务分解"
- **关键词**: 任务编排, task orchestration, 工作流自动化, 多步骤任务, 任务分解, 流程自动化

---

## 功能说明

### 核心功能

1. **任务分解**: 将复杂任务分解为可执行的子任务
2. **技能调度**: 自动调用相关 Skills 完成子任务
3. **依赖管理**: 管理子任务之间的依赖关系
4. **结果汇总**: 汇总所有子任务的结果
5. **进度跟踪**: 实时跟踪任务执行进度

### 借鉴 DeerFlow 的思想

| DeerFlow 概念 | 本项目实现 |
|--------------|----------|
| 子代理编排 | 多 Skills 协同工作 |
| 任务分解 | 智能任务分解 |
| 依赖管理 | 子任务依赖关系管理 |
| 结果汇总 | 自动汇总子任务结果 |
| 沙箱执行 | 利用 Trae IDE 的代码执行能力 |

---

## 输入输出

### 输入

- **任务描述**: 用户输入的任务描述或目标
- **任务要求**: 可选的时间要求、质量要求等
- **输出格式**: 可选的输出格式和详细程度

### 输出文件

| 输出类型 | 位置 | 文件命名 |
|---------|------|---------|
| 任务计划 | `输出成果\任务编排\` | `YYYY-MM-DD_[任务]_计划.md`|
| 执行报告 | `输出成果\任务编排\` | `YYYY-MM-DD_[任务]_报告.md`|
| 中间结果 | `输出成果\任务编排\temp\` | 临时文件|

## 工作流程

```
步骤1：理解任务需求
─────────────────────
用户输入 → 解析任务目标 → 明确任务要求

步骤2：任务分解
─────────────────────
任务目标 → 分解为子任务 → 建立依赖关系
     │
     ├── 子任务1：[描述]
     ├── 子任务2：[描述]
     ├── 子任务3：[描述]
     └── 子任务4：[描述]

步骤3：技能映射
─────────────────────
子任务列表 → 匹配 Skills → 制定执行计划
     │
     ├── 子任务1 → Skill A
     ├── 子任务2 → Skill B
     ├── 子任务3 → Skill C
     └── 子任务4 → Skill A

步骤4：执行任务
─────────────────────
执行计划 → 按依赖顺序执行 → 收集结果
     │
     ├── 执行子任务1 → 结果1
     ├── 执行子任务2 → 结果2
     ├── 执行子任务3 → 结果3
     └── 执行子任务4 → 结果4

步骤5：结果汇总
─────────────────────
所有结果 → 整合 → 生成最终结果
     │
     ├── 整合结果
     ├── 生成报告
     └── 知识沉淀

步骤6：生成报告
─────────────────────
最终结果 → 结构化输出 → 执行报告
                              ↓
          输出成果\任务编排\YYYY-MM-DD_[任务]_报告.md
```

## 任务分解策略

### 通用任务分解

```
任务类型：研究类
子任务：
1. 信息收集（深度研究 Skill）
2. 数据整理（日工作记录 Skill）
3. 分析总结（深度研究 Skill）
4. 知识沉淀（日工作记录 Skill）

依赖关系：
1 → 2 → 3 → 4
```

依赖关系：

子任务：

3. 结构化输出（日工作记录 Skill）
