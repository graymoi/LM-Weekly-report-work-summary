# 子智能体编排 Skill

## 概述

协调多个子智能体协同工作，处理复杂任务和并行工作流。

---

## 核心概念

### 什么是子智能体编排？

子智能体编排是指将复杂任务分解为多个子任务，分配给不同的子智能体并行或串行执行，最后汇总结果的过程。

### 编排模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **并行模式** | 多个子智能体同时执行 | 独立任务、快速处理 |
| **串行模式** | 子智能体按顺序执行 | 依赖任务、流程控制 |
| **混合模式** | 并行+串行组合 | 复杂任务、灵活调度 |
| **管道模式** | 前一个输出作为后一个输入 | 数据处理、转换流程 |

---

## 触发条件

### 显式触发

- "并行执行"、"同时处理"
- "子智能体"、"编排任务"
- "分解任务"、"协同工作"

### 自动触发

- 任务可分解为独立子任务
- 任务执行时间过长
- 用户请求并行处理

---

## 编排流程

```
任务分析
    │
    ├── 识别子任务
    │       ├── 独立性分析
    │       ├── 依赖关系分析
    │       └── 优先级分析
    │
    ├── 分配子智能体
    │       ├── 能力匹配
    │       ├── 负载均衡
    │       └── 资源分配
    │
    ├── 执行编排
    │       ├── 并行执行
    │       ├── 串行执行
    │       └── 混合执行
    │
    └── 结果汇总
            ├── 结果合并
            ├── 冲突解决
            └── 质量验证
```

## 子智能体类型

### 1. 研究型智能体

**职责**：信息收集、研究分析

**能力**：
- 文献检索
- 数据分析
- 趋势预测

### 2. 创作型智能体

**职责**：内容生成、创意设计

**能力**：
- 文档撰写
- 方案设计
- 创意构思

### 3. 验证型智能体

**职责**：质量检查、结果验证

**能力**：
- 逻辑验证
- 格式检查
- 一致性验证

### 4. 执行型智能体

**职责**：具体任务执行

**能力**：
- 代码生成
- 文件操作
- 系统操作

## 编排示例

### 示例1：并行研究

```
用户请求：研究三个不同领域的政策

编排方案：
┌─────────────────────────────────────┐
│           主智能体                    │
│   (任务分解 + 结果汇总)               │
└──────────────┬──────────────────────┘
               │
       ┌───────┼───────┐
       │       │       │
       ▼       ▼       ▼
   ┌─────┐ ┌─────┐ ┌─────┐
   │子智能│ │子智能│ │子智能│
   │体 A  │ │体 B  │ │体 C  │
   │财政  │ │住建  │ │发改  │
   └─────┘ └─────┘ └─────┘
       │       │       │
       └───────┼───────┘
               ▼
         汇总报告
```

               │
               ▼

              │
              ▼
   ┌─────────────────────┐

## 配置选项

在 `hook-config.yaml` 中配置：

```yaml
subagent:
  enabled: true
  max_parallel: 3
  timeout: 300
  retry_count: 2

  types:
    research:
      enabled: true
      skills: ["深度研究", "文献检索"]
    creative:
      enabled: true
      skills: ["文档撰写", "方案设计"]
    verification:
      enabled: true
      skills: ["验证循环", "质量检查"]
    execution:
      enabled: true
      skills: ["代码生成", "文件操作"]
```

## 最佳实践

- 汇总前验证子结果
- 检查一致性
- 处理冲突
