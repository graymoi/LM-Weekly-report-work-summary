# 使用统计 Skill

## 概述

自动收集和分析Skills、Hooks、Rules的使用数据，为自我进化提供数据支撑。

---

## 核心功能

### 1. 数据收集
- 收集Skills使用次数
- 收集Hooks触发次数
- 收集Rules应用次数
- 收集会话时长

### 2. 数据分析
- 分析使用趋势
- 识别高频功能
- 识别低频功能
- 发现使用模式

### 3. 报告生成
- 生成日统计报告
- 生成周统计报告
- 生成月统计报告
- 生成趋势分析报告

### 4. 进化建议
- 基于数据生成进化建议
- 识别优化机会
- 预测使用趋势

---

## 触发条件

### 显式触发

- "使用统计"、"统计数据"
- "查看统计"、"统计报告"
- "使用分析"、"数据分析"

### 自动触发

- 每日结束时（自动生成日报告）
- 每周结束时（自动生成周报告）
- 每月结束时（自动生成月报告）

---

## 统计指标

### Skills统计

| 指标 | 说明 | 计算方式 |
|------|------|---------|
| 使用次数 | Skill被调用的次数 | 计数 |
| 使用率 | 占总调用的比例 | 次数/总次数 |
| 成功率 | 执行成功的比例 | 成功次数/总次数 |
| 平均时长 | 平均执行时间 | 总时长/次数 |
| 用户满意度 | 用户反馈评分 | 平均分 |

### Hooks统计

| 指标 | 说明 | 计算方式 |
|------|------|---------|
| 触发次数 | Hook被触发的次数 | 计数 |
| 触发率 | 占总触发的比例 | 次数/总次数 |
| 执行成功率 | 执行成功的比例 | 成功次数/总次数 |
| 平均执行时间 | 平均执行耗时 | 总时长/次数 |

### Rules统计

| 指标 | 说明 | 计算方式 |
|------|------|---------|
| 应用次数 | Rule被应用的次数 | 计数 |
| 应用率 | 占总应用的比例 | 次数/总次数 |
| 冲突次数 | 与其他Rule冲突的次数 | 计数 |

### 会话统计

| 指标 | 说明 | 计算方式 |
|------|------|---------|
| 会话次数 | 总会话数 | 计数 |
| 平均时长 | 平均会话时长 | 总时长/次数 |
| 平均对话轮数 | 平均每会话对话轮数 | 总轮数/次数 |
| 知识产出 | 每会话产出的知识量 | 计数 |

## 数据存储

### 存储位置

```
输出成果/使用统计/
├── 日统计/
│   ├── 2026-03-14.md
│   └── 2026-03-15.md
├── 周统计/
│   └── 2026年第11周.md
├── 月统计/
│   └── 2026年3月.md
└── 原始数据/
    ├── skills_usage.json
    ├── hooks_usage.json
    └── sessions.json
```

### 数据格式

```json
{
  "date": "2026-03-14",
  "skills": {
    "日工作记录": {
      "count": 5,
      "success": 5,
      "failed": 0,
      "total_duration": 25.5,
      "avg_duration": 5.1
    }
  },
  "hooks": {
    "session-start": {
      "count": 3,
      "success": 3,
      "failed": 0,
      "total_duration": 1.2,
      "avg_duration": 0.4
    }
  },
  "sessions": {
    "total": 3,
    "total_duration": 180,
    "avg_duration": 60,
    "total_turns": 45,
    "avg_turns": 15
  }
}
```

## 与其他Skills协同

```
使用统计
    │
    ├──→ 技能进化
    │       └──→ 提供使用数据 → 触发进化
    │
    ├──→ 自我优化
    │       └──→ 提供优化依据 → 执行优化
    │
    └──→ 技能评估
            └──→ 提供评估数据 → 生成评估报告
```

---

## 配置选项

在 `hook-config.yaml` 中配置：

```yaml
statistics:
  enabled: true
  track_skill_usage: true
  track_hook_triggers: true
  track_session_duration: true
  output_path: "输出成果/使用统计"
```

## 最佳实践

### 1. 定期查看
- 每周查看周统计报告
- 每月查看月统计报告
- 关注趋势变化

### 2. 数据驱动
- 基于数据做决策
- 识别低频功能考虑优化
- 识别高频功能考虑增强

### 3. 隐私保护
- 不收集敏感信息
- 数据仅用于系统优化
- 支持用户删除数据
