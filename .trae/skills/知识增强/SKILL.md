# 知识增强 Skill

## 基本信息

- **名称**: 知识增强
- **功能**: 智能关联和检索，借鉴 DeerFlow 的长期记忆和上下文工程能力
- **触发词**: "知识关联"、"知识检索"、"智能查询"、"知识图谱"
- **关键词**: 知识增强, knowledge enhancement, 智能关联, 知识检索, 知识图谱, 关联分析

---

## 功能说明

### 核心功能

1. **智能关联**: 自动发现知识之间的关联关系
2. **知识检索**: 基于语义的智能知识检索
3. **关联分析**: 分析关联的强度和类型
4. **知识图谱**: 生成可视化的知识图谱
5. **知识推荐**: 基于关联关系推荐相关知识

### 借鉴 DeerFlow 的思想

| DeerFlow 概念 | 本项目实现 |
|--------------|----------|
| 长期记忆 | 知识库持久化存储 |
| 上下文工程 | 智能上下文管理 |
| 关联发现 | 自动关联发现 |
| 知识检索 | 语义检索 |

## 工作流程

```
步骤1：理解查询需求
─────────────────────
用户输入 → 解析查询内容 → 明确查询类型

步骤2：知识检索
─────────────────────
查询内容 → 搜索知识库 → 获取相关知识
     │
     ├── 项目知识库
     ├── 个人知识库
     └── 政策库

步骤3：智能关联
─────────────────────
相关知识 → 分析关联 → 发现关联关系
     │
     ├── 项目关联
     ├── 人物关联
     ├── 政策关联
     └── 主题关联

步骤4：关联分析
─────────────────────
关联关系 → 分析强度 → 分类关联类型
     │
     ├── 强关联（直接相关）
     ├── 中关联（间接相关）
     └── 弱关联（潜在相关）

步骤5：知识推荐
─────────────────────
关联结果 → 推荐相关知识 → 生成推荐列表
     │
     ├── 相关项目
     ├── 相关政策
     ├── 相关案例
    └── 相关经验

步骤6：生成报告
─────────────────────
所有结果 → 结构化输出 → 知识报告
                              ↓
          输出成果\知识增强\YYYY-MM-DD_[主题]_关联分析.md
```

## 知识检索策略

### 语义检索

```
检索方法：
1. 关键词匹配：精确匹配查询关键词
2. 语义相似：基于语义相似度检索
3. 上下文检索：基于上下文检索相关知识

检索范围：
- 项目知识库
- 个人知识库
- 政策库
```

### 关联检索

```
检索方法：
1. 直接关联：检索直接相关的知识
2. 间接关联：检索间接相关的知识
3. 潜在关联：检索潜在相关的知识

关联类型：
- 项目关联：项目之间的关联
- 人物关联：人物之间的关联
- 政策关联：政策之间的关联
- 主题关联：主题之间的关联
```

## 关联分析策略

### 关联强度分析

```
关联强度：
- 强关联（直接相关）：关联度 > 0.7
- 中关联（间接相关）：0.4 < 关联度 ≤ 0.7
- 弱关联（潜在相关）：关联度 ≤ 0.4

关联度计算：
- 关键词重叠度
- 语义相似度
- 上下文相关性
```

### 关联类型分析

```
关联类型：
1. 项目关联
   - 参考关联：项目A参考项目B
   - 协同关联：项目A与项目B协同
   - 支撑关联：项目A支撑项目B
   - 竞争关联：项目A与项目B竞争

2. 人物关联
   - 同事关联：人物A与人物B是同事
   - 合作关联：人物A与人物B有合作
   - 上下级关联：人物A是人物B的上级
   - 项目关联：人物A与人物B在同一个项目

3. 政策关联
   - 上位关联：政策A是政策B的上位政策
   - 下位关联：政策A是政策B的下位政策
   - 相关关联：政策A与政策B相关
   - 冲突关联：政策A与政策B冲突

4. 主题关联
   - 同类关联：主题A与主题B同类
   - 包含关联：主题A包含主题B
   - 被包含关联：主题A被主题B包含
   - 相关关联：主题A与主题B相关
```

## 知识图谱生成

### 图谱结构

```
节点类型：
- 项目节点：项目编号、项目名称
- 人物节点：姓名、单位、职务
- 政策节点：政策名称、发文号
- 主题节点：主题名称

边类型：
- 参考边：项目A → 项目B
- 协同边：项目A ↔ 项目B
- 支撑边：项目A → 项目B
- 合作边：人物A ↔ 人物B
- 上位边：政策A → 政策B
- 相关边：主题A ↔ 主题B
```

### 图谱可视化

```
可视化方式：
1. 节点图：显示节点和边
2. 力导向图：基于力导向布局
3. 层次图：基于层次结构
4. 时间图：基于时间序列
```
