# 深度研究 Skill

## 基本信息

- **名称**: 深度研究
- **功能**: 自动化深度研究流程，借鉴 DeerFlow 的深度研究能力
- **触发词**: "深度研究"、"研究一下"、"深入分析"、"深度探索"
- **关键词**: 深度研究, research, 深度分析, 系统研究, 全面调研, 文献综述, 专题研究

---

## 功能说明

### 核心功能

1. **多轮信息收集**: 自动进行多轮搜索，从不同角度收集信息
2. **智能整理**: 自动分类、去重、整合收集到的信息
3. **深度分析**: 提供结构化的分析和洞察
4. **知识沉淀**: 自动提取可复用的知识，更新知识库
5. **报告生成**: 生成结构化的研究报告

### 借鉴 DeerFlow 的思想

| DeerFlow 概念 | 本项目实现 |
|--------------|----------|
| 深度研究框架 | 多轮信息收集 + 智能整理 |
| 子代理协作 | 调用多个 Skills 协同工作 |
| 上下文工程 | 自动维护研究上下文 |
| 长期记忆 | 自动更新知识库 |

---

## 输入输出

### 输入

- **研究主题**: 用户输入的研究主题或问题
- **研究范围**: 可选的时间范围、地域范围等
- **输出要求**: 可选的报告格式、详细程度等

### 输出文件

| 输出类型 | 位置 | 文件命名 |
|---------|------|---------|
| 研究报告 | `输出成果\深度研究\` | `YYYY-MM-DD_[主题]_研究报告.md`|
| 知识更新 | `项目库\_项目知识\` 或 `个人知识库\` | 自动更新相应文件|

## 工作流程

```
步骤1：理解研究需求
─────────────────────
用户输入 → 解析研究主题 → 明确研究范围和目标

步骤2：制定研究计划
─────────────────────
研究主题 → 分解为子问题 → 制定搜索策略
     │
     ├── 子问题1：背景信息
     ├── 子问题2：现状分析
     ├── 子问题3：关键问题
     └── 子问题4：解决方案

步骤3：多轮信息收集
─────────────────────
子问题列表 → research-lookup → 收集信息
     │
     ├── 第一轮：背景信息
     ├── 第二轮：现状分析
     ├── 第三轮：关键问题
     └── 第四轮：解决方案

步骤4：智能整理
─────────────────────
收集的信息 → 去重 → 分类 → 整合
     │
     ├── 去除重复信息
     ├── 按主题分类
     └── 整合相关内容

步骤5：深度分析
─────────────────────
整理的信息 → 提取关键点 → 发现关联 → 生成洞察
     │
     ├── 关键点提取
     ├── 关联发现
     └── 洞察生成

步骤6：知识沉淀
─────────────────────
分析结果 → 识别知识 → 更新知识库
     │
     ├── 项目知识 → 项目库\_项目知识\
     ├── 个人知识 → 个人知识库\
     └── 政策信息 → 政策库.md

步骤7：生成报告
─────────────────────
所有结果 → 结构化输出 → 研究报告
                              ↓
          输出成果\深度研究\YYYY-MM-DD_[主题]_研究报告.md
```

## 研究策略

### 通用研究策略

```
主题分解：
1. 背景信息：基本概念、发展历程、相关政策
2. 现状分析：当前状况、主要问题、发展趋势
3. 关键问题：核心问题、影响分析、解决方案
4. 实践案例：成功案例、失败案例、经验教训
```

### 政策研究策略

```
主题分解：
1. 政策背景：政策出台的背景和目的
2. 政策内容：政策的主要内容和要求
3. 政策解读：政策的关键条款和含义
4. 政策影响：政策的影响和预期效果
5. 政策案例：政策的应用案例
```

### 技术研究策略

```
主题分解：
1. 技术概述：技术的基本概念和原理
2. 技术发展：技术的发展历程和现状
3. 技术对比：与其他技术的对比分析
4. 技术应用：技术的应用场景和案例
5. 技术趋势：技术的发展趋势和前景
```

## 知识沉淀规则

### 项目知识识别

```
识别模式：
- 案例经验：可复用的项目经验、成功/失败案例
- 政策信息：相关政策、法规、标准
- 人物信息：相关专家、决策者

处理方式：
- 新知识 → 添加到对应知识库文件
- 已有知识 → 更新或补充
```

### 个人知识识别

```
识别模式：
- 技能：新掌握的技能、方法、工具
- 经验：可复用的经验、教训、技巧
- 学习：新学习的知识点、概念

处理方式：
- 新知识 → 添加到个人知识库
- 已有知识 → 更新或补充
```

## 注意事项

1. **研究深度**: 根据主题复杂度调整研究轮次

4. **报告质量**: 确保报告结构清晰、内容完整
