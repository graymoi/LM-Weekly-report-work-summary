# 新闻监测 Skill

## 基本信息

- **名称**: 新闻监测
- **功能**: 自动化新闻监测与政策收集
- **触发词**: "/查询新闻"、"/查看新闻"、"新闻监测"
- **关键词**: 新闻监测, 政策收集, 新闻搜索, 舆情监测

---

## 功能说明

### 核心功能

1. **关键词监测**: 根据关键词库自动搜索相关新闻
2. **去重过滤**: 基于历史记录去重，避免重复收集
3. **时间过滤**: 只收集指定时间范围内的新闻
4. **结构化输出**: 生成格式化的新闻监测记录

### 适用场景

- 每日新闻监测
- 政策动态跟踪
- 行业资讯收集

## 工作流程

```
步骤1：读取关键词库
─────────────────────
新闻关注关键词.md → 解析关键词列表

步骤2：执行搜索
─────────────────────
关键词列表 → WebSearch搜索 → 获取结果

步骤3：去重过滤
─────────────────────
搜索结果 → 读取history.json → 过滤已收集

步骤4：时间过滤
─────────────────────
过滤结果 → 只保留24小时内的新闻

步骤5：生成记录
─────────────────────
最终结果 → 结构化输出
                    ↓
    输出成果\新闻监测记录\YYYY-MM-DD.md

步骤6：更新历史
─────────────────────
新收集的新闻 → 更新history.json
```

## 去重机制

### history.json 格式

```json
{
  "last_update": "2026-03-01T10:00:00",
  "news_history": [
    {
      "title": "新闻标题",
      "url": "https://...",
      "date": "2026-03-01",
      "keywords": ["关键词1", "关键词2"]
    }
  ]
}
```

### 去重规则

1. **标题去重**: 相同标题的新闻不重复收集
2. **URL去重**: 相同URL的新闻不重复收集
3. **时间限制**: 只保留最近30天的历史记录

## 注意事项

1. **搜索频率**: 建议每日执行一次，避免频繁搜索
2. **关键词维护**: 定期更新关键词库，保持监测的准确性
3. **历史清理**: history.json 会自动清理30天前的记录
