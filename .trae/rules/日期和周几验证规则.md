# 日期和周几验证规则

## 规则目的

防止工作总结中日期和周几对应错误，确保时间信息的准确性。

## 规则内容

### 1. 日期和周几对应关系表

|------|------|

### 2. 验证步骤

#### 步骤1：生成工作总结前
- 检查工作总结的日期范围
- 确认日期和周几的对应关系

#### 步骤2：生成工作总结时
- 使用JavaScript Date对象计算日期对应的周几
- 验证计算结果与对应关系表是否一致
- 如果不一致，立即停止并检查

#### 步骤3：生成工作总结后
- 逐行检查工作总结中的日期和周几
- 确保所有日期和周几的对应关系正确
- 如果发现错误，立即修正

### 3. 验证工具

#### JavaScript验证脚本

```javascript
// 日期和周几验证脚本
function validateDate(year, month, day, expectedWeekday) {
  const weekdays = ['周日', '周一', '周二', '周三', '周四', '周五', '周六'];
  const d = new Date(year, month - 1, day);
  const actualWeekday = weekdays[d.getDay()];

  console.log(`${year}年${month}月${day}日: ${actualWeekday}`);
  console.log(`预期: ${expectedWeekday}`);

  if (actualWeekday === expectedWeekday) {
    console.log('✅ 正确');
  } else {
    console.log('❌ 错误！');
  }
  console.log('---');
}

// 验证本周日期
validateDate(2026, 3, 2, '周一');
validateDate(2026, 3, 3, '周二');
validateDate(2026, 3, 4, '周三');
validateDate(2026, 3, 5, '周四');
validateDate(2026, 3, 6, '周五');
validateDate(2026, 3, 7, '周六');
validateDate(2026, 3, 8, '周日');
```

#### 检查清单

在生成工作总结时，必须检查以下内容：

- [ ] 工作总结的日期范围是否正确
- [ ] 每个日期对应的周几是否正确
- [ ] 文件名中的日期范围是否与内容一致
- [ ] 内容中的日期和周几是否对应
- [ ] 是否有日期和周几不一致的情况

### 4. 常见错误

#### 错误1：日期和周几不对应
- **错误示例**：3月2日是周日
- **正确示例**：3月2日是周一
- **原因**：没有验证日期和周几的对应关系

#### 错误2：文件名日期范围不正确
- **错误示例**：文件名是2.26-3.1，但内容中提到周一
- **正确示例**：文件名是2.24-3.1，内容中提到周一
- **原因**：文件名中的日期范围没有验证

#### 错误3：内容中的日期和周几不一致
- **错误示例**：内容中提到"周一（3月2日）"，但3月2日是周日
- **正确示例**：内容中提到"周一（3月2日）"，3月2日是周一
- **原因**：内容中的日期和周几没有验证

### 5. 注意事项

#### JavaScript Date对象注意事项

- **月份从0开始**：0=1月，1=2月，...，11=12月
- **getDay()返回值**：0=周日，1=周一，...，6=周六
- **时区问题**：JavaScript Date对象使用本地时区

#### 验证时机

- **生成前**：验证日期范围
- **生成时**：验证日期和周几对应关系
- **生成后**：逐行检查日期和周几

### 6. 应急处理

#### 发现错误时的处理步骤

1. **立即停止**：发现日期和周几不一致时，立即停止生成
2. **查找原因**：检查是文件名错误、内容错误，还是计算错误
3. **修正错误**：根据正确信息修正错误
4. **重新验证**：修正后重新验证所有日期和周几
5. **记录错误**：记录错误原因和修正方法，避免再次发生
